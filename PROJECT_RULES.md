# PROJECT_RULES

This repository contains the frontend for the SIMA demo.

## Tech stack
- React
- TypeScript
- Vite
- Tailwind CSS
- React Router

## General rules
- Keep the code modular and clean.
- Prefer reusable components.
- Do not add unnecessary libraries.
- Use functional React components only.
- Use TypeScript types when useful.
- Keep styling in Tailwind classes unless a separate file is clearly justified.
- Avoid overengineering. This is a demo.
- Do not modify project structure without a clear reason.
- Maintain a premium, minimal, futuristic UI.

## Design direction
- Dark interface
- Purple and black animated blurred background
- Elegant, modern, premium look
- Clean spacing
- Rounded elements
- Soft glassmorphism where appropriate
- Avoid cluttered dashboards
- Avoid heavy enterprise-looking tables unless necessary

## Main screens
- Login
- Home
- Meeting
- Transcript

## Demo constraints
- Frontend-first
- Some data can be mocked
- Focus on visual quality and user flow
- No complex backend integration is required unless explicitly requested
